# /ux-planning — UX Blueprint erstellen

Erstelle einen UX-Blueprint für ein Feature oder eine Änderung. Lies zuerst den vollständigen Skill und folge dessen Prozess.

## Ablauf

1. Lies `skills/ux-planning.md` und folge dem beschriebenen Prozess vollständig
2. Starte mit dem Triage (Skip / Light / Full) basierend auf `$ARGUMENTS`
3. Falls `$ARGUMENTS` leer ist, frage den User was geplant werden soll

## Kontext

Dieser Command wird typischerweise verwendet:
- Während oder nach Brainstorming, wenn UX-Entscheidungen anstehen
- Vor Implementation, um Screens und Flows zu definieren
- Wenn ein Feature zu komplex ist um direkt draufloszubauen
